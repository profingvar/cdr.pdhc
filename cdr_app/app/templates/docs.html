Architecture

cdr.pdhc is the canonical clinical data repository for the PDHC platform. It receives normalised observations from gateway.pdhc and stores them in a three-layer architecture:

LayerTable(s)Purpose
Rawingest_rawImmutable payload archive with SHA-256 hash for deduplication
Standardfhir_resources, openehr_compositionsDual-format clinical data — FHIR R5 Observations and openEHR Compositions, cross-linked
Canonicalhealth_observations, activitiesQuery-optimised flat tables with metric, value, unit, concept_guid

Supporting tables: clinical_context (provenance), dedupe_registry, audit_log, loinc_archetype_map, service_keys, users.

Data Flow

gateway.pdhc ──POST /api/v1/ingest──▶ cdr.pdhc
                                        │
         ┌──────────────────────────────┤
         ▼                              ▼
   1. Deduplicate (SHA-256)       2. Store raw (immutable)
         │
         ▼
   3. Store/transform standard layer
      ├── FHIR provided? Store + generate openEHR
      └── openEHR provided? Store + generate FHIR
         │
         ▼
   4. Store canonical (health_observations / activities)
         │
         ▼
   5. Store clinical context (careplan, plandef, transaction)
         │
         ▼
   6. Register in dedupe registry
         │
         ▼
   7. Enqueue Cambio delivery (if concept-mapped)
         │
         ▼
   8. Audit log

MethodPathDescription

Cambio CDR Sandbox Integration

Observations with a concept_guid referencing plan.pdhc.se are automatically queued for delivery to the Cambio Platform Innovation sandbox. The delivery worker runs every 60 seconds with exponential backoff retry (max 5 attempts).

Delivery flow

  1. Ingest pipeline creates cambio_delivery_log entries (status: pending)
  2. Worker ensures patient exists in Cambio (creates via FHIR Patient + openEHR EHR)
  3. Worker delivers FHIR Observation via FHIR Gateway and openEHR Composition via xCDR
  4. On success: status → delivered, cambio_resource_id stored
  5. On failure: retry with backoff (10s, 20s, 40s, 80s, 160s). After 5 failures: status → failed

Delivery scope: Only observations with canonical references to plan.pdhc.se concepts. Data without concept mappings is stored locally but not pushed upstream.
